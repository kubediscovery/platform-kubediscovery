## ADDED Requirements

### Requirement: Dual-mode analysis execution
The platform MUST support AI analysis requests that execute either in the control plane or in the data plane according to cluster configuration and policy.

#### Scenario: Route analysis to control plane analyzer
- **WHEN** a cluster is configured for server-side analysis
- **THEN** analysis requests SHALL be executed by control-plane analyzer services without requiring client-side analyzer workloads

### Requirement: Event-driven troubleshooting analysis
The system MUST support analyzer pipelines initiated from runtime events generated by data-plane components.

#### Scenario: Analyzer triggered by executor watcher event
- **WHEN** a data-plane executor emits a troubleshooting event
- **THEN** the agent SHALL forward the event to gateway analysis flow and the platform SHALL generate an analysis result record

### Requirement: Token handling boundaries
AI access tokens MUST be managed by the control plane and delegated to data-plane analysis only when explicitly allowed by configuration and policy.

#### Scenario: Client-side analysis without delegation permission
- **WHEN** a data-plane analyzer request requires token usage but delegation is not allowed
- **THEN** the platform SHALL deny client-side execution and return a policy-compliant error
