Required env vars:
```bash

DATABRICKS_ENDPOINT_NAME=databricks-claude-sonnet-4-6

# Dev: static token
```

The `databricksmodel/` package implements `model.LLM` from Google ADK (`google.golang.org/adk`), bridging ADK's genai format to Databricks' OpenAI-compatible API.

## Architecture decisions that affect all future code

- **DI**: every service uses `go.uber.org/fx`. Entry point is `cmd/grpc/main.go` (or `cmd/http/main.go`). Each domain exposes `var Module = fx.Module(...)` in `module.go`.
- **Config**: `spf13/viper`. CLI reads `~/.kubediscovery/config.yaml`; services read env vars; dev uses `.env` via `godotenv`.
- **gRPC transport**: all Control Plane ↔ Data Plane communication is **bidirectional gRPC with mTLS**. Certificates are generated by `kdctl init` and stored in `~/.kubediscovery/certs/<environment>/`.
- **Proto source of truth**: `proto/kubediscovery/v1/*.proto`. Generated Go code lives in `libs/core/v1/<service>/`. **Never edit generated `*.pb.go` files manually.** Run `make proto-gen` (or `scripts/proto-gen.sh`) after any `.proto` change.
- **Observability**: every service must expose Prometheus `/metrics` and emit OpenTelemetry traces (OTLP HTTP exporter). Traces start at each gRPC/HTTP handler and propagate via gRPC metadata.

## kd-gateway: critical identity logic

The gateway identifies agents from **three sources in priority order**:
1. `caller_id` field (logical, self-reported — used as map key in memory)
2. `metadata` (free `google.protobuf.Struct` — logged but not trusted for auth)
3. mTLS certificate CN/SAN (trusted — issued by CA from `kdctl init`)

`GRPC_MTLS=1` (default) requires `GRPC_CLIENT_CA_FILE` pointing to the CA cert.  
`GRPC_DEBUG=1` enables per-call duration interceptors.

## kd-agent: critical env vars

| Var | Default | Note |
|---|---|---|
| `AGENT_ID` | `kd-agent` | **Must be unique per agent instance** — becomes `caller_id` |
| `GRPC_ADDR` | `localhost:50051` | kd-gateway address |
| `GRPC_CA_FILE` | `~/.kubediscovery/certs/staging/ca.crt` | Validates server cert |
| `GRPC_CLIENT_CERT_FILE` | `~/.kubediscovery/certs/staging/srv004.crt` | Agent mTLS cert |
| `GRPC_CLIENT_KEY_FILE` | `~/.kubediscovery/certs/staging/srv004.key` | Agent mTLS key |

Retry loop: exponential backoff, base 1s, weight 3, max 5 attempts (delays: 1s, 3s, 9s, 27s, 81s → fatal).

## kd-analyzer: LLM pipeline

- Uses **Google ADK** (`google.golang.org/adk v1.2.0`) as agent orchestrator.
- Each agent runs in an **isolated session** — no shared history between agents.
- Output of one agent is passed manually as input to the next (Go pipeline, not ADK multi-agent).
- `karpenter_analyst` is **conditional**: only triggered when pod is `Pending` AND events contain `"untolerated taint"`, `"0/N nodes available"`, or `"FailedScheduling"`.
- Always returns `AnalysisResult` with `MemoryKey = clusterName+environment+namespace` for kd-store indexing.
- Model per agent is configured via env vars (`MODEL_LOG_ANALYST`, `MODEL_EVENT_ANALYST`, etc.).

## Certificate workflow (kdctl)

```bash
# 1. Initialize CA + server cert (interactive TUI or flags)
kdctl init --name <name> --address <host:port> --environment <env>
# Writes to: ~/.kubediscovery/certs/<environment>/ca.crt, ca.key, server.crt, server.key

# 2. Generate client cert for kd-agent
kdctl certificate --create --name client-<agent> --environment <env> --client
# Auto-detects --client if name starts with "client-"
# CA must already exist (created by kdctl init)

# 3. List all certs
kdctl certificate --list
```
